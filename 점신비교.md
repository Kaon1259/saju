# 점신 vs 1:1연애 — 오늘의 운세 개선 로드맵

비교 기준일: 2026-04-19 / 비교 대상: 점신(1900만 명 플랫폼)

---

## 1. 완료 (이번 커밋)

### ✅ 시간대별 운세 (hourlyFortune)
점신의 핵심 차별점이었던 "시간대별 길흉"을 도입.

- 5구간: 아침(06-10시) / 점심(11-14시) / 오후(14-18시) / 저녁(18-22시) / 밤(22-02시)
- 각 구간: `time`, `range`, `score`(45-98), `desc`(한 줄 조언)
- 서버 반영:
  - `FortunePromptBuilder` — JSON 스키마에 `hourlyFortune[]` 요구
  - `DailyFortune.hourlyFortuneJson` TEXT 컬럼
  - `FortuneService.parseAndSaveStreamResult` — 파싱·캐시·복원
  - `SajuResult.CategoryFortune.hourlyFortune` — partner/other 경로
- 클라이언트:
  - `HourlyTimeline` 컴포넌트 (아이콘·점수 바 그래프·순차 페이드인)
  - mine/partner/other 3탭 전부 반영
  - 라이트모드 대응

## 2. 미완료 — 우선순위순

### 🟡 2순위: 방위/방향운 (luckyDirection)
**왜 필요한가**: 점신이 회의·이동·약속 있는 날 실사용 가치를 만드는 요소. 구현 비용도 작음.

- JSON에 `"luckyDirection": "동쪽(진방)"` 필드 1개 추가
- 프롬프트: "오늘 길한 방위와 이유를 한 줄로" 지시
- DailyFortune에 `lucky_direction` VARCHAR 컬럼
- 클라이언트: `myf-lucky` 카드에 방향 블록 1칸 추가 (숫자·색 옆에 🧭 아이콘)

**예상 작업량**: 서버 1h + 클라이언트 30m

---

### 🟡 3순위: 행운 아이템 확장 (food, fashion, item)
**왜 필요한가**: 점신의 추천소식(음식/패션/아이템) 섹션에 대응. 실생활 연결고리 증가 → 재방문 유도.

- JSON 필드 3개 추가:
  - `luckyFood`: "따뜻한 차 또는 매콤한 국밥"
  - `luckyFashion`: "네이비 셔츠에 실버 액세서리"
  - `luckyItem`: "가죽 지갑 또는 만년필"
- `혈액형 운세` 프롬프트에는 이미 `food`/`direction` 있으니 공통 CategoryFortune 필드로 승격 가능
- 클라이언트: "오늘의 행운 아이템" 2x2 그리드 카드 신규

**예상 작업량**: 서버 45m + 클라이언트 1h + 아이콘/그리드 CSS

---

### 🟠 4순위: 바이오리듬 통합 노출
**왜 필요한가**: 점신은 운세보고서 1화면에 바이오리듬 포함. 현재는 별도 페이지 `/biorhythm`에 분리되어 접근성 ↓.

- MyFortune 결과 하단에 "오늘의 바이오리듬" 미니 차트 카드 추가
- 기존 `BiorhythmService` 결과를 재사용 (추가 AI 호출 없음)
- 신체/감정/지성 3가지 게이지만 컴팩트하게
- 클릭 시 `/biorhythm` 전체 페이지로 이동

**예상 작업량**: 클라이언트 1.5h (신규 작은 컴포넌트 + API 호출 1개)

---

### 🟢 5순위(선택): 별점 UX
**왜 필요한가**: 점신은 별점(★), 우리는 100점. 한눈에 파악되는 직관성은 별점 쪽이 우세.

- 데이터는 100점 그대로 유지 (라디오 그래프용)
- 점수 옆에 `★★★★☆` 5단계 변환 표시 (90+ = 5별 / 75+ = 4별 / …)
- 메인 원형 점수 + 보조 별점 병행

**예상 작업량**: 클라이언트 30m (변환 함수 + 렌더)

## 3. 구현 시 주의사항

### 캐시/히스토리 복원 일관성
- 새 필드 추가 시 반드시 3곳 모두 반영:
  1. `FortunePromptBuilder` JSON 스키마
  2. `FortuneService.parseAndSaveStreamResult` + `DailyFortune` 엔티티
  3. `SajuService.parseAndSaveStreamResult` + `SajuResult.CategoryFortune` (partner/other 경로)
- 누락 시 mine 탭만 동작하고 partner/other에서 안 보이는 버그 발생

### Railway 배포
- DDL auto-update가 신규 컬럼(`hourly_fortune_json`, `lucky_direction`, …) 자동 추가
- 서버 코드 수정 후 즉시 `git push` → Railway 재배포 필요
  (`feedback_server_deploy.md` 규칙)

### 토큰 비용
- 현재 `maxTokens=1500` → 필드 추가마다 응답 길이 증가
- hourlyFortune 추가로 약 +200 토큰 예상 (Sonnet 4.6 기준 비용 증가 미미)
- 방향·음식·패션·아이템까지 합치면 +100 토큰 추가
- 필요 시 `maxTokens=1800` 상향

## 4. 권장 진행 순서

1. **이번 커밋 배포 검증** (Railway 반영 + 실제 AI 응답이 hourlyFortune 5구간 전부 채우는지 확인)
2. **2순위 방위/방향운** — 가장 ROI 높음, 구현 작음
3. **3순위 행운 아이템 확장** — 방위와 같은 카드 영역에 자연스럽게 추가
4. **4순위 바이오리듬 통합** — 작업량 큼, 마지막에
5. **5순위 별점** — 다른 작업 중 곁들이기 가능

## 5. 우리 앱이 점신보다 우위인 것 (유지 강점)

- **3탭 구조** (내 / 연인 / 다른 사람) — 점신은 본인만
- **날짜 확장** (오늘 / 내일 / 임의 날짜) — 점신은 오늘 고정
- **SSE 스트리밍 타이핑** — 점신은 정적 텍스트
- **20대 친화 대화체** — 점신은 보고서 문체
- **히스토리 드로어** (cacheOnly 선조회) — 점신은 단일 세션
- **AI 심화분석 옵션** — 점신은 유료 상담 상품
- **타로 78장 멀티 덱** — 점신은 제한적
