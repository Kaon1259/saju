# 사주운세 앱 - 서비스 구성 가이드

## 아키텍처

```
[사용자 브라우저]
       |
       v
[Vercel] ── 프론트엔드 (React/Vite)
  saju.vercel.app
       |
       | HTTPS API 호출
       v
[Render] ── 백엔드 (Spring Boot)
  saju-server.onrender.com
       |
       v
[TiDB Cloud] ── 데이터베이스 (MySQL 호환)
       |
       v
[Claude API] ── AI 운세 생성
```

---

## 1. 데이터베이스 - TiDB Cloud (무료)

### 가입 및 생성
1. https://tidbcloud.com 접속 → 회원가입
2. **Create Cluster** → **Serverless** 선택 (무료)
3. Region: `Asia Pacific (Tokyo)` 선택 (한국에서 가장 가까움)
4. Cluster 이름: `saju-db`
5. 생성 완료 후 **Connect** 클릭

### 연결 정보 확인
- Connect 화면에서 아래 정보를 메모:
  - **Host**: `gateway01.xxx.prod.aws.tidbcloud.com`
  - **Port**: `4000`
  - **User**: `xxxxx.root`
  - **Password**: 생성 시 설정한 비밀번호

### DB 초기화
- TiDB 콘솔(SQL Editor)에서 실행:
```sql
CREATE DATABASE IF NOT EXISTS saju_db;
```
- 테이블은 Spring Boot가 자동 생성 (`ddl-auto: update`)

## 2. 백엔드 - Render (무료)

### 가입 및 배포
1. https://render.com 접속 → **GitHub로 가입**
2. **New** → **Web Service**
3. GitHub repo 연결: `Kaon1259/saju`
4. 설정:
   - **Name**: `saju-server`
   - **Root Directory**: `server`
   - **Runtime**: `Docker`
   - **Instance Type**: `Free`

### 환경 변수 설정
| Key | Value | 설명 |
|-----|-------|------|
| `DB_URL` | `jdbc:mysql://호스트:4000/saju_db?sslMode=VERIFY_IDENTITY` | TiDB 연결 URL |
| `DB_USERNAME` | `xxxxx.root` | TiDB 유저명 |
| `DB_PASSWORD` | `비밀번호` | TiDB 비밀번호 |
| `CLAUDE_API_KEY` | `sk-ant-api03-...` | Claude API 키 |
| `CLAUDE_MODEL` | `claude-haiku-4-5-20251001` | 사용할 모델 (선택) |
| `CLOVA_CLIENT_ID` | TTS 클라이언트 ID | 네이버 TTS (선택) |
| `CLOVA_CLIENT_SECRET` | TTS 시크릿 | 네이버 TTS (선택) |

### 배포 결과
- URL: `https://saju-server.onrender.com`
- 참고: 무료 티어는 15분 미사용 시 슬립 → 첫 요청 30~60초 대기

---

## 3. 프론트엔드 - Vercel (무료)

### 가입 및 배포
1. https://vercel.com 접속 → **GitHub로 가입**
2. **Add New Project** → `Kaon1259/saju` 선택
3. 설정:
   - **Root Directory**: `client`
   - **Framework Preset**: `Vite`
   - **Build Command**: `npm run build` (자동 감지)
   - **Output Directory**: `dist` (자동 감지)

### 환경 변수 설정
| Key | Value | 설명 |
|-----|-------|------|
| `VITE_API_URL` | `https://saju-server.onrender.com/api` | 백엔드 API 주소 |

### 배포 결과
- URL: `https://saju.vercel.app` (또는 자동 생성된 URL)

---

## 배포 순서 요약

```
1. TiDB Cloud 가입 → 클러스터 생성 → DB 연결 정보 확보
2. Render 가입 → Web Service 생성 → 환경 변수 입력 → 배포
3. Render URL 확인 (예: saju-server.onrender.com)
4. Vercel 가입 → Project 생성 → VITE_API_URL에 Render URL 입력 → 배포
5. Vercel URL로 접속 확인
```

---

## 주요 파일

| 파일 | 용도 |
|------|------|
| `server/Dockerfile` | Render Docker 빌드 설정 |
| `server/src/main/resources/application-prod.yml` | 운영 환경 설정 (환경변수 기반) |
| `server/src/main/java/.../config/WebConfig.java` | CORS 허용 도메인 |
| `client/src/api/fortune.js` | API baseURL (VITE_API_URL 환경변수) |
| `server/src/main/resources/schema.sql` | DB 스키마 (참고용, 자동 생성됨) |

---

## 커스텀 도메인 연결 (선택)

도메인 구매 후 (예: `saju-fortune.com`):
- Vercel: Settings → Domains → 도메인 추가 → DNS 설정
- Render: Settings → Custom Domains → 도메인 추가
- WebConfig.java CORS에 해당 도메인 추가 필요
