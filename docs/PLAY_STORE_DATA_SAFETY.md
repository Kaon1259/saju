# Google Play Store — Data Safety 신고 가이드

**대상 앱**: 1:1연애 (com.love.onetoone)
**작성일**: 2026-04-30
**현행 권한**: `INTERNET` 만 (AdMob 도입 전 기준)

Play Console > 앱 콘텐츠 > 데이터 보안(Data Safety) 항목에 그대로 입력하면 되도록 정리.

---

## 0. 사전 답변

### 데이터를 수집·공유합니까?
**예** — 카카오 로그인 + 프로필 입력 + 운세 분석 시 AI 처리 위탁.

### 모든 데이터가 전송 중 암호화됩니까?
**예** — 모든 API 통신은 HTTPS (Railway TLS, 카카오 OAuth HTTPS). Capacitor `androidScheme: "https"` 사용.

### 사용자가 데이터 삭제를 요청할 수 있습니까?
**예** — 마이메뉴 > 프로필 > 회원 탈퇴 버튼 (`DELETE /api/users/{id}`)
- 외부 삭제 요청 URL: 제공 가능 (`https://yourdomain.com/legal/account-deletion` 페이지 별도 필요)

### Google Play 가족 정책 대상 앱입니까?
**아니오** — 13세 이상 대상 (운세/연애 콘텐츠).

## 3. 데이터 처리 위탁 명시 (Privacy Policy 5조와 일치)

### Anthropic, PBC (Claude AI)
- **전송 데이터**: 생년월일, 출생 시간, 성별, 혈액형, MBTI, 연인 정보, 사용자가 입력한 꿈/심리테스트 답변
- **전송 안 하는 데이터**: 이름(닉네임 가능), 이메일, 카카오 ID, IP, 디바이스 정보
- **목적**: AI 운세/궁합/꿈해몽/심리테스트 결과 생성
- **보관**: Anthropic 측 서비스 정책 준수 (입력은 모델 학습에 사용되지 않음)

### Kakao Corp.
- **전송 데이터**: OAuth 인가코드 → 액세스 토큰 → 카카오 회원번호·닉네임·프로필 이미지(있는 경우)
- **목적**: 간편 로그인 인증
- **보관**: Kakao 서비스 정책

### Railway (Inc.)
- **호스팅 위탁만 — Play Store 정의상 "공유" 아님**
- 모든 사용자 데이터는 Railway MySQL DB 에 저장 (TLS at rest 옵션 적용)

## 4. 출시 전 추가 체크리스트

- [ ] **공개 개인정보처리방침 URL 등록** — `https://yourdomain.com/legal/privacy-v1.md` 또는 별도 정적 페이지
- [ ] **계정 삭제 URL** — Play Console 별도 필드 (앱 외부에서도 삭제 요청 가능한 경로 안내)
  - 옵션 A: 약관 페이지에 "회원 탈퇴는 앱 마이메뉴 > 프로필 > 회원 탈퇴 버튼" 안내 + 이메일 (kaon1259@naver.com)
  - 옵션 B: 별도 웹페이지 (`/legal/account-deletion.html`) 신규
- [ ] **AdMob 도입 시**: AndroidManifest.xml 에 `<uses-permission android:name="com.google.android.gms.permission.AD_ID" />` 추가 + Data Safety "광고 ID" 항목 "예"로 변경
- [ ] **콘텐츠 등급(Content Rating)** — IARC 설문: 운세/연애 콘텐츠 → 12+ 권장 (Teen / Mature 17+ 아님)
- [ ] **App Content > Target Audience**: 13세 이상
- [ ] **Closed Testing**: 신규 개인 개발자 계정은 12명 이상 / 14일 이상 폐쇄 테스트 통과 후 프로덕션 가능
- [ ] **App Bundle 서명 키**: Play App Signing 권장 (구글이 서명 키 보관)

## 5. 자주 걸리는 거부 사유 (사전 차단)

### "콘텐츠 정책 - 의료/건강 주장"
- 운세 결과가 "당신은 우울증입니다" 같은 진단형 표현을 출력하면 위반
- **현재 안전**: 프롬프트가 "오락 목적 참고" 톤 + 약관 5조 면책

### "데이터 보안 정책 - 민감 데이터 평문"
- application.yml 의 평문 API 키(Kakao/Claude)가 Play Store 직접 거부 사유는 아니나, git push 노출은 별도 보안 사고
- **권장**: Railway env 로 분리 (이미 부분 적용)

### "기만적 행위"
- 무료 앱이라 광고하면서 광고 미신고 시 거부
- **현재 안전**: 광고 미도입

### "권한 남용"
- INTERNET 만 사용 중 → 안전
- AdMob 도입 시 AD_ID 권한 추가 필요
